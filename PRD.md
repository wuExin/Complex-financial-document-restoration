# 复杂金融文档还原挑战 PRD

| 项目 | 内容 |

## 2. 背景与问题

金融行业沉淀了大量保险条款、费率表、批注、合同、产品说明等核心业务文档。这类文档通常具备以下特点：

- 内容密度高：正文、编号、脚注、表格、批注、特殊符号混排。
- 版式复杂：多级标题、多栏排版、嵌套表格、跨页章节、跨块图表。
- 图像规格极端：存在长宽比超过 30 的长图，以及单页像素超过 2 亿的超大图片。
- 专业语境强：包含保险责任、费率、精算、备案号、条款编号等领域文本。

传统 OCR 容易割裂语义，传统版面分析方法在非标准排版下鲁棒性不足。赛题要求在受限模型与算力条件下，构建具备感知、切块、重组、去重与纠错能力的多模态文档解析 Workflow，将复杂金融文档截图还原为与原文严格对应的 Markdown 文本。

## 3. 产品目标

构建一个端到端文档解析系统，对输入测试集图片自动完成解析，并生成符合赛事提交规范的 `submission.csv`。

核心目标：

| 目标 | 说明 |
| --- | --- |
| 内容完备性 | 覆盖原始图片中的所有有效文本、表格、脚注和特殊符号，避免漏检、错检和幻觉。 |
| 结构高保真 | 准确还原 `#` 至 `######` 标题层级、段落、列表、Markdown 表格和章节结构。 |
| 阅读流正确 | 正确处理多栏排版、跨块图表、嵌套内容和跨页章节，输出符合人类阅读顺序的文本。 |
| 工程鲁棒性 | 面对超大图、密集表格、长文档、错行目录、切块边界重复等情况稳定运行。 |
| 复现合规 | B 榜 Top15 阶段可在主办方封闭环境中 3 小时内稳定复现提交结果。 |

## 4. 范围说明

### 4.1 本期范围

- 支持读取赛事发布的图片文件夹。
- 支持超大图自动切块，避免 OOM。
- 支持调用 FinixDoc-VL API 完成局部图像解析。
- 支持切块结果的阅读流排序、拼接、去重、纠错。
- 支持 Markdown 标准化输出。
- 支持生成符合赛事要求的 CSV 文件。
- 支持一键运行和复现。
- 支持运行日志、失败重试和中间结果缓存。

### 4.2 非本期范围

- 不提供 Web 可视化标注平台。
- 不训练或集成超过 10M 参数的本地模型。
- 不调用除 FinixDoc-VL 外的任何大模型或第三方 VLM API。
- 不针对测试集图片进行硬编码结果填充。

## 5. 用户与使用场景

### 5.1 参赛开发者

作为参赛开发者，希望通过一键命令对测试集图片进行批量解析，并得到可直接提交的 CSV 文件。

典型流程：

1. 准备测试集图片目录。
2. 配置 FinixDoc-VL API 凭证和运行参数。
3. 执行 `python main.py --input_dir data/test --output submission.csv`。
4. 检查日志和输出文件。
5. 上传 CSV 到赛事平台。

### 5.2 算法工程师

作为算法工程师，希望能够调试切块策略、Prompt、去重策略和后处理规则，持续优化 A/B 榜分数。

典型流程：

1. 使用训练集图片和 GT 运行本地评估。
2. 对比预测 Markdown 与 GT 差异。
3. 调整切块、排序、拼接、表格修复策略。
4. 回归测试，确保改动没有引入明显退化。

### 5.3 评审复现人员

作为评审复现人员，希望在封闭环境中根据复现文档执行一条命令，稳定复现选手 B 榜提交结果。

典型流程：

1. 解压工程代码包。
2. 安装依赖。
3. 配置官方 API。
4. 执行一键脚本。
5. 校验生成结果与 B 榜提交文件一致。

## 6. 数据与提交规范

### 6.1 输入数据

赛事数据以文件夹形式发布，包含原始图片和对应 GT 标签文件。

评测阶段：

| 阶段 | 数据 | 用途 |
| --- | --- | --- |
| A 榜 | 训练集 200 条，测试集 100 条 | 算法验证、参数调优、阶段筛选 |
| B 榜 | 全新测试集 100 条 | 最终闭源评测 |

### 6.2 输出数据

系统输出标准 CSV 文件，建议命名：

- A 榜：`predict_A.csv`
- B 榜：`predict_B.csv`
- 通用：`submission.csv`

字段要求：

| 字段名称 | 是否必填 | 类型 | 说明 |
| --- | --- | --- | --- |
| `file_name` | 是 | String | 测试集图片文件名，含扩展名，例如 `doc_001.png` |
| `ground_truth` | 是 | String | 系统生成的标准 Markdown 结构化字符串 |

约束：

- CSV 必须包含且仅包含 `file_name`、`ground_truth` 两列。
- 必须包含表头。
- 编码必须为 UTF-8。
- 行数必须与测试集图片数量完全一致。
- 字段内换行符必须正确转义，避免 CSV 错位。

## 9. 系统流程设计

整体流程：

```text
输入图片目录
  -> 图片扫描与元数据读取
  -> 图像预处理
  -> 智能切块
  -> FinixDoc-VL 局部解析
  -> 局部 Markdown 缓存
  -> 阅读流排序
  -> 跨块拼接
  -> 去重与纠错
  -> Markdown 标准化
  -> CSV 校验与输出
```

建议模块：

| 模块 | 职责 |
| --- | --- |
| `image_loader` | 图片发现、尺寸读取、内存安全加载。 |
| `preprocess` | 图像增强、旋转校正、必要的裁边。 |
| `chunker` | 切块策略、重叠区域、坐标记录。 |
| `vl_client` | FinixDoc-VL API 调用、重试、限流、缓存。 |
| `prompt` | Prompt 模板和场景化指令管理。 |
| `layout_order` | 阅读流排序、多栏识别、块序重排。 |
| `merge` | 跨块 Markdown 合并。 |
| `dedup` | 重叠文本去重、边界重复消除。 |
| `table_repair` | Markdown 表格格式修复。 |
| `exporter` | CSV 生成和提交文件校验。 |
| `evaluator` | 训练集本地对比、差异分析和指标近似评估。 |

## 10. Prompt 需求

Prompt 应满足以下约束：

- 只输出图片中真实存在的内容。
- 不总结、不解释、不补充、不翻译。
- 按人类阅读顺序输出。
- 使用标准 Markdown。
- 标题使用 `#` 至 `######` 表示层级。
- 表格使用 Markdown 表格输出。
- 保留条款编号、备案号、标点、特殊符号和脚注。
- 对无法识别的极少量字符应尽量根据视觉内容还原，禁止凭空编造。

示例方向：

```text
你是金融保险文档解析助手。请将图片中的所有可见内容严格转写为 Markdown。
要求：
1. 不要总结、解释或补充图片中不存在的内容。
2. 保留原始文字、编号、标点、特殊符号。
3. 按真实阅读顺序输出。
4. 标题使用 Markdown 标题层级。
5. 表格使用 Markdown 表格，尽量保持单元格结构。
6. 若图片是切块，请只输出该切块中可见的完整内容，不重复输出明显来自相邻区域的残缺内容。
```

## 11. 评分指标与验收标准

线上综合评分满分 100 分：

```text
Overall = ((1 - Text Edit) * 100 + Table TEDS + (1 - Read Order Edit) * 100) / 3
```

指标解释：

| 指标 | 目标 |
| --- | --- |
| Text Edit | 降低字级 Levenshtein 编辑距离，减少漏字、错字、重复、幻觉。 |
| Table TEDS | 提升表格结构和单元格文本还原准确率。 |
| Read Order Edit | 降低逻辑块顺序编辑距离，保证多栏和跨块阅读流正确。 |

验收标准：

| 验收项 | 标准 |
| --- | --- |
| 提交文件格式 | CSV 字段、编码、行数全部符合赛事要求。 |
| 内容覆盖 | 训练集抽样检查无大段漏检。 |
| 表格输出 | 密集表格可形成合法 Markdown 表格。 |
| 阅读顺序 | 多栏样例阅读顺序符合人工判断。 |
| 去重效果 | 切块接缝处无明显重复段落。 |
| 工程复现 | 一键脚本可完整跑通，运行时间不超过 3 小时。 |
| 合规审计 | 无违规模型 API、无大模型硬编码调用、无测试集结果硬编码。 |

## 12. 合规要求

最终提交工程必须满足：

- 核心算法与 Workflow 使用 Python 编写。
- 仅允许调用主办方指定的 FinixDoc-VL API。
- 可使用完全本地离线、无需 GPU、参数量小于 10M 的轻量辅助模型。
- 不得接入任何其他第三方大模型 API。
- 不得使用超过限制的大型本地模型。
- 不得对测试集图片硬编码结果。
- 不得包含随机填充、无依据猜测或面向榜单的作弊逻辑。

开发阶段可使用 ChatGPT、通义千问等工具辅助写代码和构思方案，但不得进入最终提交工程的运行依赖链路。

## 14. 风险与应对

| 风险 | 影响 | 应对 |
| --- | --- | --- |
| 超大图 OOM | 任务中断，无法提交完整结果 | 使用分块读取、金字塔缩放、坐标切片和内存监控。 |
| 切块边界截断 | 文字缺失或重复 | 设置重叠区域，结合文本相似度去重和边界合并。 |

| 表格结构丢失 | Table TEDS 变差 | 对表格区域使用更细粒度切块和表格后处理修复。 |
| API 不稳定 | 运行超时或结果缺失 | 增加重试、缓存、断点续跑和失败兜底。 |
| Prompt 输出不稳定 | 复现不一致 | 固定 Prompt、温度参数和后处理规则。 |
| 合规审计失败 | 成绩取消 | 隔离外部模型依赖，保留依赖清单和调用审计日志。 |
